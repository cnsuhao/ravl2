These classes provide a machine-independent interface to allow the creation and management of threads within an application.

For thread-safe operation, the following are the responsibility of the programmer:

  • All shared memory access must be locked, e.g. using MutexC or MutexLockC
  • All variables of built-in type in shared memory must be declared volatile
