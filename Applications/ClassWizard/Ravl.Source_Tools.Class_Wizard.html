Class wizard is program designed to help with writing and maintaining Ravl style 'big objects'. You write the body class, exactly as it should eventually be:

  • with class name ending ...BodyC
  • with an entry under HEADERS in the defs.mk file

If you run cwiz (with no options) it will add the handle class to all appropriately-formatted body classes that it finds, in that directory and any nested directories. By default, the results are written to a file with name prepended with "_".

Notes:

  • Uses the public part of the body class as reference to maintain the handle class.
  • Modifies existing header files, no need for seperate header compilation stage. Default currently prefixes modified file with an underscore, use -p "" to replace original file.
  • Works around any modifications made by the author.
  • All methods and classes that are added are marked with '//!author:cwiz'

Operations:

  • Create handle a method for each public body method.
  • Create handle constructors for each public constructor in the body class.
  • If no handle class is found, creates the handle class and places it after the body class.
  • Remove handle methods and constructors for which there are no body methods. (Only methods marked with '//!author:cwiz')
  • Optionally will create stub functions for the .cc file with '-s' option.
