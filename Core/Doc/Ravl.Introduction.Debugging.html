This will eventually be a serious page on how to debug with RAVL. For the moment, here is a small list of tips when looking at data in ddd, the GUI for the GNU debugger gdb:

  • Printing salars, or scalar array members, is straightforward. Thus if you have an array SArray1dC a(3), you can look at an element: 
    print a[2]
  • Printing the contents of complex data structures is not so straightforward at present. If you print the array - 
    print a
    - you get the contents of the handle: 
    $1 = {> = {> = {buff = 0x805f8b8}, 
range = {minI = {v = 0}, maxI = {v = 2}}},
buff = { >> = { >> = {body = 0x805f8a0},
}, }}
    However, if you open the Data->Memory window from ddd, and fill in the number of elements and the buffer adress from the "print a" results, you get the array contents.
  • If you want to know the contents of a  StringC mystring: 
    (gdb) print (char*) mystring.rep.Where->s
  • It does seem that you can use at least some of the methods of a class in a print statement. Unfortunately it just doesn't understand "cout <<". Neither does it understand the RAVL Save() function - this has a meaning withing gdb.
