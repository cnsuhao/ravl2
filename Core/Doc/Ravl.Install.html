This document provides a guideline for installation, but has not been tested thoroughly. The instructions are for a Linux environment, using a C shell. However RAVL should also compile under M$ Windows, provided that a compiler is used that is broadly ANSI compatible.

The installation process by default will build several versions of the library, including varying degrees of optimisation and debug information, and also including shared and static versions.

RAVL Prerequisites

The RAVL library requires the use of the gcc compiler, version 4.0.2 or later. If it is not on the system, install it first.

The gtk library is strongly recommended. Much of RAVL will not build if gtk is not installed.

Installing the RAVL library and applications

Currently we recommend that you don't log in as root for this installation. Decide on a place for the installation and define the environment variable RAVL_INSTALLATION accordingly, e.g.:

setenv RAVL_INSTALLATION ~/src

(If using a bash shell, use export instead of setenv.)

The RAVL library and executables can then be installed as follows (selecting the relevant config.local.xxx file if not a standard Linux 32-bit installation):

mkdir -p $RAVL_INSTALLATION
cd $RAVL_INSTALLATION
svn co https://svn.sourceforge.net/svnroot/ravl/main/Ravl
cp Ravl/QMake/config.local.linux .

Then edit the chosen config.local.xxx. Typical modifications include:

  • turning off some of the library versions via the FULLBUILD_TARGETS variable (especially if space is limited - a full install currently uses about 2Gbyte):
    • debug - includes debugging information and array bound checking
    • check - includes array bound checking
    • opt - generates optimised code with no array bound checking
    • debugshared, checkshared, optshared - builds shared versions of the above. These link faster
    • optbin - builds application binaries - including CxxDoc, the program that builds this documentation. So if you turn optbin off, you need to build CxxDoc separately to get a complete installation.
  • picking up the correct compiler if a new one has been installed, or incorporating ccache
  • adjusting the available resources using the RESOURCES variable. The packages in the Ravl/Contrib directory all depend on various external libraries; different operating systems have different sets of these libraries. Most of them are turned off by default. It is probably best to leave them turned off when first installing RAVL. Also if your system doesn't have for example gtk, needed by the RAVL GUI module, that needs turning off. Finally, the installation is:

cd $RAVL_INSTALLATION/Ravl
./install $RAVL_INSTALLATION/RavlProject auto $RAVL_INSTALLATION/config.local.linux
rehash

(The rehash is only necessary if using a C-shell.)

Usage

Put the following alias and variable settings for compilation (modified if appropriate) in your .cshrc file:

setenv RAVL_INSTALLATION ~/src
setenv PROJECT_OUT ~/ProjOut
alias qm 'make -f $RAVL_INSTALLATION/RavlProject/share/RAVL/QMake/QMake.mk USERBUILD=1'
setenv PATH = ($PATH:$PROJECT_OUT/lib/RAVL/linux/bin:$RAVL_INSTALLATION/RavlProject/bin)
  • RAVL_INSTALLATION must be set as above.
  • $PROJECT_OUT controls the directory into which your own code will be compiled.
  • qm is the alias for building your own code. Type qm help for options for qm.
  • The additions to $PATH will include your own executables built using qm, as well as some RAVL utilities.

Locally-built documentation for the RAVL classes can then be found at: $RAVL_INSTALLATION/RavlProject/share/doc/RAVL/Auto/Basic/Tree/Ravl.html
