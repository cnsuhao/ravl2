How To Use RAVL

RAVL contains a lot of useful class libraries and a fair selection of executable packages. This page gives pointers to information on using the ready built software packages and the file formats that are supported in the group. It will also direct you to reference information about what functionality is supported by the classes and instructions on using them.

An online Tutorial is provided to help you get started with programming with RAVL. It covers setting up your environment variables and then illustrates most of the significant ways of working with the RAVL libraries using several simple examples. The next step would be to be aware of some Programming Notes that apply when using RAVL.

Once you are ready to write your own programs using RAVL you will need to be able to find out what functionality already exists in RAVL (that might solve your problem) and how to use it. This is, after all, one of the main purposes of RAVL. There are several ways of accessing what is available and it is up to you to experiment with the different representations and find out what works for you. If you have a good idea that will make the information clearer, don't hesitate to send a Proposal.

Reference information on RAVL can be accessed in the following ways:

  • Beginners might be interested in a list of Basic Classes, but this list is very limited.
  • An automatically-generated Class Tree provides a list of all base classes and shows a complete tree of behaviour derivation. A web page describing each class is available. Included is automatically generated documentation which includes programmers usage comments as well as a complete list of member functions, the necessary include file and library dependency.
  • There are a lot of functions in various namespaces. You can find some rudimentary help in the (rather large) namespace page.
  • To get more information on using the supplied packages see the pages on Executables.
  • A (fairly useless) RAVL Search engine will look for keywords in the class pages of the automatically generated documentation.
  • An RAVL Source Tree will give access to the complete source code directory tree. Help in navigating through this tree can be found in the header files. E.g. if you wanted to find the source code for MatrixC, first click on the link "amma/Matrix.hh" near the top of the page. Near the top of the file is the line: 
    //! file="amma/NumAlg/LinAlg/Matrix.hh"
    The path following "amma/" indicates the path to take through the Source Tree.
  • There is also a bit of info on obscure File Formats.

There is a class wizard program, classwiz, to help you write RAVL classes. It can generate big and small objects, and creates an appropriate defs.mk file, or updates an existing one (if checked out). There is no documentation - just run it and experiment with the options in the GUI.

Bugs: Unfortunately the tool that automatically generates the Class Tree documentation does not generate documentation on anything else, e.g. namespaces, typedefs, global functions, #define macros, ..... Therefore there is virtually no documentation of any kind on these features.
