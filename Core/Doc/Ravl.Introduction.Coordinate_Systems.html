In the image processing community the usual convention is that, in a 2-D image coordinate system, the x axis is pointing horizontally to the right, and the y axis is pointing vertically downwards, i.e. a clockwise coordinate system.

On the other hand, 2-D RAVL objects such as arrays, matrices and images, the indices are in the order: row, column (as in conventional matrix notation). In pairs of reals (e.g. 2-D vectors and points), the components are in the order x, y. Normally these two categories of objects do not interact. However, when they do (e.g. if we want to construct an Index2dC pixel position from a Point2dC position vector), the outcome in RAVL is this:

  • The row index is associated with the x value
  • The column index is associated with the y value

The implication is that RAVL adopts a 2-D coordinate system in which the x coordinate points downwards, and the y coordinate to the right. The opposite of what you were expecting? It was done like this with the best of intentions to create a consistent system. It has the advantages that:

  • the coordinate system is anti-clockwise, as is conventional in mathematics
  • the first array coordinate is associated with the first vector component etc.

Thus you can always make these associations in RAVL: "x, y" ⇔ "vector[0], vector[1]" ⇔ "rows, columns" ⇔ "down, across".

However we recognise that this convention has confused almost everyone. We decided that changing x and y was too dangerous, so our compromise has been to remove the symbols x and y wherever possible and replace them with "row" and "col".
