There are a few typedef's commonly used in RAVL code. These make it easier to port code transparently between different platforms as it allows you to make assumptions about types you can't make about the standard 'C' types. e.g. In RAVL IntT is guarantee'd to always be at least a 32-bit interger.

Its also worth noting the IndexC class, this has been introduced to as a work around for the many problems that can arrise from C's rounding and sign conversion rules. For more details see IndexC's documentation.

Following is a summary of the most commonly used typedef's:
  1. IntT - A signed interger with at least 32 bit representation.
  2. UIntT - An unsigned interger with at least 32 bit representation.
  3. ByteT - An unsigned char of at least 8 bits.
  4. SByteT - A signed char of at least 8 bits.
  5. Int16T - An signed integer of exactly 16 bits.
  6. UInt16T - An unsigned integer of exactly 16 bits.
  7. RealT - A floating point value of at least 64 bits and that provides fast arithmetic.
  8. RealT - A floating point value of at least 32 bits.
  9. SizeT - An unsigned integer which enough precision to represent the size of the largest array that can be created in a program on the machine.
