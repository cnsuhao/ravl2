This page describes the two basic ways extra documentation can be added outside the class pages:

  • Extra documentation to the documentation branch nodes - see example (also the page you are reading).
  • Stand-alone pages - see example. (The page you are reading probably should have been here.)

Adding extra documentation to the branch nodes

The "branch nodes" are those whose URL (like this page) end as: .../Tree/xxx.html. You create them like this:

  • Create a normal HTML file (suffix .html). Add author and docentry lines: 

        <!--! author="Robert Crida" -->
    <!--! docentry="Ravl.Documentation.Sections" -->

    The docentry entry determines where the page goes in the documentation tree.

    The CxxDoc system will add: some standard links; links to the parents of the node, and a heading that is duplicated from the title in the HTML head. If an automatically-generated branch node page already exists at the same URL, it will be merged - see example.

  • Add an EHT entry to the defs.mk file.

Stand-alone pages

If you have information that does not obviously fit in the class documentation tree, or is not HTML, it can be included as a stand-alone page - it can be .html, .pdf, .png, .txt etc. No formatting is done. It should be included as an HTML item in the defs.mk file (even if ti is not specifically an HTML file. It is linked from tree or class nodes as e.g. ../../../html/xxx.html (see e.g. link to "complex Gabor filter theory" on this page)
